- (version) Verified (mid-2026): Next.js 16.2.x (App Router default, Turbopack, Route Handlers app/api/*/route.ts, Node 20+, create-next-app@latest). Claude models: claude-haiku-4-5 ($1/$5) for explanations, claude-sonnet-5 richer; SDK @anthropic-ai/sdk. AWS Amplify Hosting has native Next.js SSR + Route 53 domain + auto HTTPS.
- (decision) AD-1 One Next.js 16 app, TS end-to-end, Route Handlers; no separate service. Prevents needless 2nd runtime/ops surface.
- (decision) AD-2 Scorer = one pure server-side module (lib/scorer); UI only renders its output. Prevents two sources of truth drifting.
- (decision) AD-3 Data joined OFFLINE (BLS+O*NET+Eloundou on SOC-2018 crosswalk) -> committed data/data.json; runtime only reads it. Prevents in-request joins.
- (decision) AD-4 LLM (claude-haiku-4-5) called server-side with computed score+components; returns explanation only, never computes score. Prevents thin-wrapper.
- (decision) AD-5 Free-text = data not instructions: system-prompt instructions + delimited user block + input length cap + per-IP/session rate limit + server-only secrets. Prevents prompt injection/cost abuse/key leak.
- (decision) AD-6 Stateless: no accounts/DB/persistence beyond request. Prevents unneeded data-governance surface.
- (decision) AD-7 Deploy AWS Amplify Hosting + Route 53 (careerstar.ashleylibasci.com) + auto HTTPS; secrets via Amplify env vars. Prevents ops sprawl/insecure config.
- (event) spine finalized; interview visual architecture.html produced. Worked in party mode (Winston/Amelia/Vex walk-on), fast path.
